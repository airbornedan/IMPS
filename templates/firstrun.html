IMPS uses a configuration file (config.ini) for many of its settings, including the database connection, server settings, and file directories. The file is in the same directory where you installed IMPS, most likely in /var/www/html if you are using Apache and MySQL on Linux. You will need to edit this file so that IMPS will work correctly.

Settings

The config.ini file contains several sections that you must set for IMPS to work.

DATABASE CONNECTION

Configure your database connection here. Set:

  • host to the host IP address
  • database to the database name
  • user to a user with
    • CREATE
    • DELETE
    • INSERT
    • SELECT
    • UPDATE
    privileges on the database.
  • password to the password for the user

DIRECTORIES

Configure your file directories here. Set:

  • item_image_dir to the directory where IMPS should store item images. The default is /static/images/items, but you may use any directory that IMPS have read/write access to.
  • image_dir to the directory where IMPS should access the program images. The The default is /static/images, but you may use any directory that IMPS have read/write access to.
  • backup_dir to directory where IMPS will store database backups. The default is /var/www/html/static/backups, but you may use any directory that IMPS have read/write access to.
